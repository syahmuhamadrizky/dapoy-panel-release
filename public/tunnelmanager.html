📚 Dokumen Panduan Deployment Klien

                
            

            

                
Karena server Anda tidak menggunakan IP Publik langsung (dilindungi
                    Cloudflare Tunnel), berikut adalah 3 metode yang dapat digunakan oleh klien untuk mendeploy website
                    mereka. Berikan akun (Username, Password, URL Webhook) kepada klien Anda beserta panduan ini.


                

                    

                        Metode
                            1
                        GitHub Auto-Deploy (Webhook) — Paling Disarankan
                    

                    
Cocok untuk website
                            standar (PHP, HTML, Laravel, dll). Ini adalah cara paling instan. Begitu ada kode baru
                        di GitHub, VPS akan otomatis menarik (PULL) file tersebut.

                    
    
                        
  1. Buka repository GitHub klien > Settings > Webhooks > Add
                                webhook.
    2. 
                        
  2. Paste URL Webhook GitHub (Bisa dicopy dari tombol 'Lihat Detail' akun).
    3. 
                        
  3. Pilih Content type: application/json (atau form-urlencoded) dan simpan.
    4. 
                        
  4. Cerdas: Folder web klien tidak perlu
                            diinisialisasi Git manual. Dapoy Panel akan otomatis men-clone / menginisialisasi Git saat
                            webhook pertama kali dipanggil!
    5. 
                    

                


                

                    

                        Metode
                            2
                        GitHub Actions (Jalur SFTP) — Untuk Framework Berat
                    

                    
Cocok untuk web
                            React/Vue/Node.js yang butuh compile (npm run build). Biarkan server canggih
                        GitHub yang mem-build aplikasinya, lalu hasil matangnya (PUSH) dikirim ke VPS Anda
                        melewati Cloudflare.

                    

                        Tambahkan step ini di file .github/workflows/deploy.yml klien tepat sebelum step
                        upload SFTP:
                        
- name: Buka Jalur Cloudflare Tunnel
  run: |
    wget -q https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb
    sudo dpkg -i cloudflared-linux-amd64.deb
    cloudflared access tcp --hostname ftp.dapoy.net --url localhost:2222 &
    sleep 3

                        Lalu set target Host SFTP di script upload GitHub Action ke localhost dan port
                        2222.
                    

                


                

                    

                        Metode
                            3
                        Transfer Manual (FileZilla / SFTP)
                    

                    
Klien harus mengaktifkan
                        koneksi tunnel di komputernya sebelum bisa buka FileZilla.

                    
    
                        
  1. Install aplikasi cloudflared di laptop/PC klien.
                        
    2. 
                        
  2. Buka CMD/Terminal di laptop klien, jalankan: 
    
                            cloudflared access tcp --hostname ftp.dapoy.net --url localhost:2222
                        
    3. 
                        
  3. Biarkan CMD terbuka. Buka FileZilla, isi Host: localhost, Port: 2222, dan
                            masukkan Username & Password akun SFTP klien.
    4.
